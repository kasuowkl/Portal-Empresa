📋 Agenda Projetos

Modelo completo para gestão hierárquica de projetos, subprojetos, tarefas, subtarefas e atividades.
Baseado na estrutura existente da Agenda de Tarefas, com extensões para colaboração avançada e controle de tempo.

📦 1. Entidades Principais

Projeto Principal

Contêiner de nível superior. Pode conter subprojetos ou tarefas diretamente.

id nome datas dono progresso

Subprojeto

Projeto filho. Herda permissões do pai ou pode ter permissões independentes.

projeto_pai_id status

Tarefa

Unidade de trabalho, pode ser dividida em subtarefas. Possui responsável, prazo, prioridade.

estimativa horas dependências status

Subtarefa

Subdivisão de uma tarefa, com conclusão individual.

tarefa_pai_id ordem

Atividade

Registro de trabalho realizado, com horas apontadas e descrição.

data_início duração anexos

Marco (Milestone)

Evento importante que agrupa tarefas e serve como ponto de controle.

data_prevista data_realizada

Comentário & Anexo

Comunicação e arquivos vinculados a qualquer entidade.

Log de Atividades

Registro automático de todas as ações para auditoria.

🔗 2. Relacionamentos (Estrutura Hierárquica)

Projeto Principal → (0..n) Subprojeto
Projeto Principal / Subprojeto → (0..n) Tarefa
Tarefa → (0..n) Subtarefa
Tarefa / Subtarefa → (0..n) Atividade
Tarefa / Projeto / Subtarefa → (0..n) Comentário + Anexo
Projeto → (0..n) Marco
Usuário → responsável por tarefas / executor de atividades / autor de comentários.

⚙️ 3. Funções por Entidade

📁 Projeto Principal e Subprojeto

  • Criar: Definir nome, datas, equipe, cor. Ex: “Lançamento App Versão 2.0” (01/04 a 30/06).
  • Editar: Alterar atributos, adicionar/remover membros, mudar cor.
  • Arquivar: Mover para projetos concluídos (sem exclusão).
  • Excluir: Remover permanentemente (confirmação obrigatória).
  • Visualizar: Painel com resumo, gráfico de progresso, marcos.
  • Gerenciar Membros: Convidar pessoas com permissões específicas (dono, editor, colaborador, leitor).

✅ Tarefa

  • Criar: Título, descrição, responsável, prazo, prioridade, categoria, estimativa de horas.
  • Editar: Alterar qualquer campo, inclusive dependências (ex: tarefa A depende de tarefa B).
  • Mover: Transferir entre projetos/subprojetos.
  • Alterar Status: A Fazer → Em Andamento → Concluída (via botões ou Kanban).
  • Registrar Horas: Lançar tempo trabalhado via atividade (iniciar/pausar/encerrar).
  • Adicionar Comentário: Inserir observações, @menções (ex: “@joao, valide esta correção”).
  • Anexar Arquivos: Upload de documentos, imagens, wireframes.
  • Gerenciar Subtarefas: Criar, concluir, reordenar.

📌 Subtarefa

  • Criar: Dentro de uma tarefa, definir subtarefa com título.
  • Marcar Concluída: Checkbox que atualiza progresso da tarefa pai.
  • Editar/Excluir: Modificar título ou remover.

⏱️ Atividade

  • Iniciar / Pausar / Encerrar: Cronometragem em tempo real.
  • Adicionar Manual: Lançar horas retroativas com descrição.
  • Editar: Corrigir descrição ou duração.

🏁 Marco

  • Criar: Definir marco com data prevista e tarefas associadas.
  • Concluir: Registrar data de realização e comentário.
  • Associar Tarefas: Vincular tarefas que devem ser concluídas para o marco ser atingido.

💬 Comentário e Anexo

  • Comentar: Texto, menções, anexos inline.
  • Editar/Excluir: Modificar ou remover comentário próprio.
  • Anexar: Upload de arquivo em qualquer entidade (tarefa, projeto, atividade).

📜 Log de Atividades

  • Registrar Automático: Captura ações como criação, edição, exclusão, mudança de status.
  • Filtrar: Por entidade, data, usuário (ex: logs do projeto “App” nos últimos 7 dias).

📘 4. Exemplo de Uso – Projeto “Sistema de Gestão de Vendas”

Projeto Principal: Sistema de Gestão de Vendas (01/04 a 31/08, dono: Carlos)
Subprojeto 1: Backend (API e banco)
  → Tarefa 1.1: Modelagem de dados
    ▪ Subtarefa 1.1.1: Criar diagrama ER
    ▪ Subtarefa 1.1.2: Implementar migrações
    ▪ Atividade: 2h – “Levantamento de requisitos”
  → Tarefa 1.2: Endpoints de autenticação
Subprojeto 2: Frontend (React)
  → Tarefa 2.1: Estrutura de componentes
Marco 1: MVP funcional (15/06) – associado às tarefas 1.1, 1.2, 2.1
Marco 2: Versão final (31/08)

Fluxo: Carlos cria projeto e subprojetos → adiciona Ana (backend) e Pedro (frontend) → Ana cria tarefa e subtarefas → Ana inicia atividade, registra 2h → Carlos acompanha progresso no painel → Pedro comenta pedindo documentação → sistema registra logs.

🔄 6. Integração com a Agenda de Tarefas Atual

  • As listas existentes serão convertidas em projetos pessoais ou subprojetos dentro de um projeto principal “Meus Projetos”.
  • Tarefas atuais mantêm status, prioridade, categoria e são vinculadas ao novo modelo.
  • Compartilhamento de listas é mapeado para permissões de projeto (dono/edição/leitura → papéis equivalentes).
  • Visualizações Kanban e Lista permanecem disponíveis, agora filtradas por projeto.

🧩 7. Considerações Técnicas

  • Banco de dados: Novas tabelas projetos (com campo parent_id para hierarquia), marcos, atividades, comentarios, anexos, logs. Tabela tarefas ganha projeto_id e campos de estimativa/horas/dependências.
  • API RESTful: Endpoints para cada entidade com validação de permissões via token JWT.
  • Frontend: Componentes reutilizáveis (árvore de projetos, visão Gantt, timeline de atividades). Manter a identidade visual do sistema original.

Comentários e Anexos

  • Comentar com @menções, anexar arquivos em qualquer entidade.

Logs

  • Registro automático de criação, edição, exclusão, mudança de status.

Estrutura de Banco de Dados (Nomes Reduzidos)

Tabelas principais para implementação:

  • projetos – auto-relacionamento (projeto_pai_id), dono_id, datas, status, cor.
  • marcos – projeto_id, nome, data_prevista, data_realizada.
  • tarefas – projeto_id, responsavel_id, prazo, prioridade, status, estimativa_horas, horas_trabalhadas.
  • subtarefas – tarefa_id, titulo, concluida, ordem.
  • atividades – tarefa_id (opcional), subtarefa_id (opcional), usuario_id, data_inicio, data_fim, duracao_minutos, descricao.
  • comentarios – entidade_tipo, entidade_id, usuario_id, conteudo.
  • anexos – entidade_tipo, entidade_id, nome_arquivo, caminho, tamanho, uploaded_por.
  • logs – usuario_id, acao, entidade_tipo, entidade_id, detalhes (JSON), data_hora.
  • membros_projeto – projeto_id, usuario_id, papel.
  • categorias (opcional) – projeto_id, nome, cor.
Exemplo SQL (MySQL) para tabelas principais: 
  CREATE TABLE projetos (
      id INT AUTO_INCREMENT PRIMARY KEY,
      nome VARCHAR(200) NOT NULL,
      descricao TEXT,
      projeto_pai_id INT NULL,
      dono_id INT NOT NULL,
      data_inicio DATE,
      data_fim DATE,
      status VARCHAR(20) DEFAULT 'planejado',
      cor VARCHAR(7),
      created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
      updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
      FOREIGN KEY (projeto_pai_id) REFERENCES projetos(id) ON DELETE CASCADE
  );
  
  CREATE TABLE tarefas (
      id INT AUTO_INCREMENT PRIMARY KEY,
      projeto_id INT NOT NULL,
      titulo VARCHAR(255) NOT NULL,
      descricao TEXT,
      responsavel_id INT,
      prazo DATE,
      prioridade VARCHAR(10),
      categoria VARCHAR(50),
      status VARCHAR(20) DEFAULT 'a_fazer',
      estimativa_horas DECIMAL(6,2),
      horas_trabalhadas DECIMAL(6,2) DEFAULT 0,
      created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
      updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
      FOREIGN KEY (projeto_id) REFERENCES projetos(id) ON DELETE CASCADE
  );
  
  -- demais tabelas seguem padrão semelhante

As listas existentes na Agenda de Tarefas podem ser convertidas em projetos, e as tarefas migradas para a nova estrutura mantendo histórico.
