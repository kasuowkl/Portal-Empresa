Versionamento e Backup
Guia visual para trabalhar com Git local, branches de seguranca e backup automatico antes de publicar no servidor do Portal.

Visao Geral

O Portal agora usa uma combinacao de Git local, branch por mudanca maior e backup automatico no servidor antes de cada publicacao manual.

Git

Historico local

Permite comparar, voltar arquivos e recuperar commits com seguranca.

Branches

Mudancas isoladas

Cada ajuste maior pode nascer numa branch propria, reduzindo risco.

Backup remoto

Versao anterior publicada

Antes de subir um arquivo, o servidor guarda a copia anterior com timestamp.

Esse modelo e mais seguro do que renomear arquivos para index_v1, index_v2 e assim por diante, porque preserva historico local e copia publicada de forma organizada.

Estrutura Recomendada

master feature/* fix/* _backups

Padrao adotado

  • master para a linha mais estavel do projeto.
  • feature/* para funcionalidades novas.
  • fix/* para correcoes pontuais.
  • _backups no servidor para guardar a versao publicada anterior.

Branches

Foi criado um script para padronizar a abertura de branches de trabalho.

.\scripts\new-work-branch.ps1 -Name "agenda-contabil-notificacoes"

Exemplo para correcao:

.\scripts\new-work-branch.ps1 -Type fix -Name "editor-colagem-texto"
O script bloqueia criacao de branch se houver alteracoes locais pendentes. Isso ajuda a evitar misturar trabalho antigo com mudanca nova.

Publicacao com Backup

Antes de publicar um arquivo no Ubuntu, use o script de publicacao com backup. Ele cria a pasta, copia a versao remota atual e so depois envia o arquivo novo.

.\scripts\publish-with-backup.ps1 `
  -LocalPath "public/baseConhecimento/index.html" `
  -RemotePath "/var/www/html/wkl/Portal/public/baseConhecimento/index.html"

Se quiser reiniciar o Portal em seguida:

.\scripts\publish-with-backup.ps1 `
  -LocalPath "routes/conhecimento.js" `
  -RemotePath "/var/www/html/wkl/Portal/routes/conhecimento.js" `
  -RestartPortal

Resultado esperado

O backup anterior passa a existir com timestamp, por exemplo:

/var/www/html/wkl/Portal/_backups/public/baseConhecimento/2026-04-02_153000__index.html

Restauracao

Se algo quebrar em producao, o restore pode ser feito direto a partir de um backup publicado.

.\scripts\restore-from-backup.ps1 `
  -BackupPath "/var/www/html/wkl/Portal/_backups/public/baseConhecimento/2026-04-02_153000__index.html" `
  -TargetPath "/var/www/html/wkl/Portal/public/baseConhecimento/index.html" `
  -RestartPortal
Esse restore e mais confiavel do que tentar lembrar "qual era o index_v3 certo", porque o arquivo mostra data e hora exatas da publicacao anterior.

Fluxo do Dia a Dia

1
Abra uma branch nova para mudancas medias ou grandes.
2
Trabalhe com commits pequenos e mensagens claras.
3
Antes de subir arquivo no servidor, use publish-with-backup.ps1.
4
Se houver problema, use restore-from-backup.ps1 para voltar.

git status

git log --oneline --decorate -n 15
