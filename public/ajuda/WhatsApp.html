Integração WhatsApp
Aprove, reprove e consulte solicitações do Portal WKL diretamente pelo WhatsApp, com login automático por número de telefone.

Visão Geral

A integração WhatsApp conecta o Portal WKL ao aplicativo WhatsApp, permitindo que os aprovadores interajam com o sistema de aprovações sem precisar acessar o portal pelo navegador.

O que você pode fazer pelo WhatsApp

  • Receber notificações de aprovações pendentes
  • Consultar aprovações pendentes, aprovadas e reprovadas
  • Aprovar ou reprovar solicitações respondendo uma mensagem
  • Ver detalhes completos de uma aprovação pelo número
  • Acessar automaticamente com seu número de telefone cadastrado
O sistema foi desenvolvido pela equipe WKL e roda em 3 serviços independentes integrados ao Portal WKL.

Arquitetura

O sistema é composto por 3 serviços que se comunicam via HTTP:

Portal WKL — porta 3000 Sistema principal. Banco de dados SQL Server. Gerencia usuários, aprovações e todos os módulos.
↕ API REST + chave de integração
WhatsApp UI — porta 3001 Conexão com o WhatsApp via Baileys. Interface web de conversas. Encaminha mensagens recebidas para o Bot.
↕ HTTP interno
WhatsApp Bot — porta 3002 Processa comandos dos usuários. Consulta o Portal. Envia respostas de volta pelo WhatsApp UI.
O serviço WhatsApp UI (3001) mantém a sessão do WhatsApp e não deve ser reiniciado desnecessariamente. O Bot (3002) e o Portal (3000) podem ser reiniciados livremente.

Autenticação — Login Automático

Quando um usuário envia uma mensagem, o bot identifica automaticamente seu login no Portal pelo número de telefone cadastrado. Não é necessário digitar usuário nem senha.

Como cadastrar seu número no Portal

1
Acesse o Portal WKL → Configurações → Usuários Locais (ou Usuários de Domínio)
2
Clique em Editar no seu usuário
3
Preencha o campo WhatsApp com seu número completo (somente dígitos, com DDI e DDD).
Exemplo: 5592991002823 → 55 (Brasil) + 92 (DDD) + 991002823 (número)
4
Clique em Salvar. Na próxima mensagem enviada ao bot, o login será automático.

Login manual (alternativo)

Se seu número não estiver cadastrado, você pode fazer login manualmente digitando:

login <usuario>
Vincula seu login do Portal à sessão atual.
Exemplo: login joao.silva
O login manual não persiste entre reinícios do bot. Para uso contínuo, cadastre o número no Portal.

Comandos disponíveis

Envie as mensagens abaixo como texto normal no WhatsApp para o número da empresa:

oi / olá / bom dia
Saudação — exibe boas-vindas e o menu principal (se autenticado)
0 / menu / ajuda
Menu principal — exibe todas as opções disponíveis
1 / pendentes
Aprovações pendentes — lista até 10 aprovações que aguardam sua decisão
2
Últimas aprovadas — lista as 10 aprovações mais recentes com status Aprovado
3
Últimas reprovadas — lista as 10 aprovações mais recentes com status Reprovado
4
Resumo geral — exibe totais de pendentes, aprovadas e reprovadas

Detalhar — exibe todas as informações de uma aprovação específica
Exemplo: detalhar 42
6 <id> / aprovar <id>
Aprovar — registra sua aprovação para a solicitação informada
Exemplo: aprovar 42
7 <id> [motivo] / reprovar <id> [motivo]

Exemplo: reprovar 42 Fora do orçamento aprovado

Aprovações no Portal

O módulo de Aprovações do Portal WKL está totalmente integrado ao WhatsApp. Toda interação — criar, aprovar ou reprovar — gera notificações automáticas para os aprovadores que tiverem número cadastrado.

Fluxo de notificações

  • Ao criar uma solicitação: cada aprovador com número cadastrado recebe uma mensagem WhatsApp com os dados da solicitação e os comandos para responder
  • Ao votar pelo WhatsApp: o voto é registrado no Portal e o histórico da solicitação é atualizado com a indicação "(via WhatsApp)"
  • Resultado dos envios: cada tentativa de envio é registrada nos Logs de Atividade (ação NOTIF_WHATSAPP) — com sucesso, erro ou aviso de número não cadastrado

Votar pelo WhatsApp

O aprovador pode responder diretamente pelo WhatsApp sem acessar o portal:

aprovar <id>
Registra aprovação. Exemplo: aprovar 42
reprovar <id> [motivo]
Registra reprovação com motivo opcional. Exemplo: reprovar 42 Fora do orçamento

Reenviar WhatsApp (Admin)

Administradores podem reenviar a notificação WhatsApp manualmente para os aprovadores que ainda não votaram, diretamente do painel de detalhes da solicitação no Portal.

Como usar

1
Acesse /aprovacoes no Portal e abra uma solicitação com status Pendente.
2
No painel de detalhes, clique no botão Reenviar WhatsApp (visível apenas para administradores).
3
Confirme a ação. O sistema envia um lembrete para todos os aprovadores com decisão ainda Pendente que tiverem número cadastrado.
4
Um toast confirma quantos foram notificados e quantos não tinham número. O resultado detalhado aparece nos Logs de Atividade.
O botão Reenviar WhatsApp só aparece para usuários com nível Admin e apenas em solicitações com status Pendente.
Use o reenvio quando um aprovador relatar que não recebeu a notificação original ou quando o bot WhatsApp estava offline no momento da criação da solicitação.

Painel Administrativo

Acesse em http://servidor:3002/admin — exclusivo para administradores do sistema.

Recursos disponíveis

  • Logs em tempo real — todas as mensagens recebidas, envios, autenticações e erros via Socket.IO
  • Sessões ativas — visualiza quais números de WhatsApp estão autenticados no bot no momento
  • Configurações do bot — editar textos das mensagens (saudação, menu, erros) sem precisar reiniciar
  • Reset de configurações — restaura todos os textos ao padrão original
  • Limpar sessões Signal — remove arquivos de sessão corrompidos sem novo QR Code

Configuração do Portal

Para que a integração funcione, o Portal WKL precisa estar configurado corretamente:

Pré-requisitos no Portal

  • Banco de dados na versão v2.2.0+ — colunas whatsapp nas tabelas usuarios e usuarios_dominio
  • Middleware de autenticação v1.1.0+ com suporte à chave de API (x-api-key)
  • Endpoint GET /api/usuarios/por-whatsapp/:numero disponível na API do Portal

Variável de ambiente no Portal (.env)

VariávelDescriçãoExemplo
WHATSAPP_API_KEYChave secreta compartilhada entre Portal e botwkl-whatsapp-2026
O valor de WHATSAPP_API_KEY (Portal) deve ser idêntico ao PORTAL_API_KEY no arquivo .env do serviço WhatsApp. Após alterar qualquer .env, reinicie com pm2 restart --update-env.

Problemas Comuns

Mensagem "número não vinculado ao Portal"

  • Verifique se o campo WhatsApp está preenchido no usuário do Portal (somente dígitos, com DDI 55)
  • Para usuários do domínio AD: clique em Atualizar do AD no cadastro para sincronizar o número
  • Confirme que PORTAL_API_KEY e WHATSAPP_API_KEY têm o mesmo valor nos dois .env
  • Reinicie o bot com pm2 restart whatsapp-bot --update-env

Mensagens ficam "Aguardando" no iPhone (não entregues)

  • Este problema é causado por incompatibilidade de sessão entre o Baileys e dispositivos iPhone (formato LID)
  • Solução: verifique se a versão do Baileys é 7.0.0-rc.9 ou superior. Versões anteriores (ex: 6.7.21) não são compatíveis com o novo protocolo LID do iPhone
  • Se necessário, atualize: npm install @whiskeysockets/baileys@7.0.0-rc.9 na pasta do serviço WhatsApp
  • Após atualizar, faça um reset completo da sessão (apague a pasta auth_info_baileys/) e escaneie o QR Code novamente

Bot não responde às mensagens

  • Confirme que todos os serviços estão online: pm2 list
  • Verifique se BOT_URL=http://localhost:3002 está correto no .env
  • Consulte os logs: pm2 logs whatsapp-wkl --lines 30

Aprovador não recebeu notificação WhatsApp ao criar solicitação

  • Acesse Configurações → Logs, filtre por Sistema: Aprovações e ação NOTIF_WHATSAPP
  • O log mostra se o envio foi bem-sucedido, se houve erro de rede ou se o número não estava cadastrado
  • Se o bot estava offline no momento da criação, use o botão Reenviar WhatsApp (disponível para admins na tela de detalhes da solicitação)

QR Code não aparece / WhatsApp desconectado

  • Acesse http://servidor:3001/admin e clique em Limpar Sessões Signal
  • Se necessário, apague toda a pasta auth_info_baileys/ e escaneie o QR Code novamente

Erro 401 ao consultar aprovações

  • As chaves PORTAL_API_KEY e WHATSAPP_API_KEY estão divergentes
  • Após corrigir, reinicie: pm2 restart portal-wkl whatsapp-bot --update-env
Em caso de dúvida, consulte sempre os logs: Painel Admin → Logs em tempo real (bot) e Configurações → Logs → NOTIF_WHATSAPP (Portal). Juntos mostram exatamente o que aconteceu em cada etapa do envio.
