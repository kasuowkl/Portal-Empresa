WhatsApp — Configuração e Logs
Autenticação do bot, comandos disponíveis, painel admin, variáveis de ambiente e guia técnico de integração.

Arquitetura dos serviços

O sistema roda em 3 serviços PM2 independentes que se comunicam via HTTP:

ServiçoPortaPM2Responsabilidade
Portal WKL3132portalSistema principal, banco de dados, APIs REST, serve /whatsapp
WhatsApp UI3200whatsappConexão Baileys, recepção/envio de mensagens, interface web
WhatsApp Bot3210whatsapp-portalInterpretação de comandos, chamadas ao Portal, telas de aprovações e admin
O serviço whatsapp (3200) mantém a sessão do WhatsApp. Reiniciá-lo exige escanear o QR Code novamente. Os demais podem ser reiniciados livremente.

Autenticação do bot

Quando um usuário envia uma mensagem, o bot identifica automaticamente seu login pelo número de telefone cadastrado no Portal.

Login automático

O bot extrai o número do JID do WhatsApp e consulta GET /api/usuarios/por-whatsapp/:numero no Portal (com API key). Se encontrado, o login é definido automaticamente para a sessão.

Login manual (alternativo)

Se o número não estiver cadastrado, o usuário pode enviar:

login <usuario>
Vincula o login do Portal à sessão atual.
Exemplo: login joao.silva
O login manual não persiste entre reinícios do bot. Para uso contínuo, cadastre o número em Configurações → Usuários → campo WhatsApp.

Painel Administrativo

Acesse em /whatsapp/admin dentro do Portal. Exclusivo para administradores.

Recursos

  • Logs em tempo real — todas as mensagens recebidas, envios, autenticações e erros via Socket.IO
  • Sessões ativas — quais números de WhatsApp estão autenticados no bot agora

Problemas Comuns

Bot não responde às mensagens

  • Confirme que todos os serviços estão online: pm2 list
  • Verifique se BOT_URL=http://localhost:3210 está correto no .env
  • Consulte os logs: pm2 logs whatsapp --lines 30

"Número não vinculado ao Portal"

  • Verifique se o campo WhatsApp está preenchido (somente dígitos, DDI 55)
  • Confirme que PORTAL_API_KEY e WHATSAPP_API_KEY são iguais nos dois .env
  • Reinicie: pm2 restart whatsapp-portal --update-env

QR Code não aparece / WhatsApp desconectado

  • Acesse /whatsapp/admin e clique em Limpar Sessões Signal
  • Se necessário, acesse diretamente http://192.168.0.80:3200, clique em Desconectar e escaneie o QR Code

Mensagens e botões não aparecem na interface

  • Verifique se o serviço WhatsApp UI (porta 3200) está rodando: pm2 list
  • Abra o console do navegador (F12) e verifique erros de rede nas chamadas para a porta 3200
  • O middleware CORS deve estar ativo em ambos os serviços — confira whatsApp.js e whatsapp-portal.js

Busca de contatos mostra lista incompleta

  • O endpoint /api/usuarios/lista-whatsapp consolida usuários locais, AD e agenda de contatos
  • Usuários sem número cadastrado não aparecem — verifique o campo WhatsApp no cadastro
Sempre consulte os logs primeiro: /whatsapp/admin → Logs em tempo real. O log mostra exatamente o que aconteceu, qual número foi buscado e qual erro retornou.

Guia técnico — CORS

O Portal serve a interface em /whatsapp (porta 3132). Essa interface faz chamadas fetch() aos backends nas portas 3200 e 3210. Sem CORS o browser bloqueia essas chamadas.

Ambos os serviços precisam do middleware abaixo antes de qualquer rota:

// whatsApp.js e whatsapp-portal.js
app.use((_req, res, next) => {
  res.header('Access-Control-Allow-Origin', '*');
  res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS');
  res.header('Access-Control-Allow-Headers', 'Content-Type');
  if (_req.method === 'OPTIONS') return res.sendStatus(204);
  next();
});

O Socket.IO também precisa de CORS explícito:

const io = new Server(server, {
  cors: { origin: '*', methods: ['GET', 'POST'] }
});

Guia técnico — Middleware do Portal

middleware/verificarLogin.js foi atualizado para aceitar chamadas de serviços externos via chave de API:

// Bloco adicionado ao verificarLogin.js
const apiKey       = req.headers['x-api-key'];
const serviceLogin = req.query._whatsapp_login || req.body?._whatsapp_login;
const chaveCorreta = process.env.WHATSAPP_API_KEY;

if (apiKey && chaveCorreta && apiKey === chaveCorreta && serviceLogin) {
  req.session.usuario = { usuario: serviceLogin, nivel: 'usuario' };
  return next();
}

Guia técnico — Endpoints criados no Portal

GET /api/usuarios/por-whatsapp/:numero

Busca login pelo número do WhatsApp (auto-login do bot). Aceita somente API key, sem _whatsapp_login. Consulta usuarios e usuarios_dominio.

GET /api/usuarios/lista-whatsapp

Retorna todos os usuários e contatos com número de WhatsApp. Consolida via UNION:

  • usuarios — campo whatsapp
  • usuarios_dominio — campo whatsapp
  • contatos — COALESCE(whatsapp, cel_pessoal, cel_corporativo)

Guia técnico — Integrar novo módulo ao bot

O padrão é sempre: Portal expõe API → portalApi.js cria função → messageHandler.js cria comando.

Use verificarLogin normalmente no endpoint — o middleware já aceita a API key do bot automaticamente.

1. Expor endpoint no Portal

// routes/chamados.js
router.get('/api/chamados/meus', verificarLogin, async (req, res) => {
  const usuario = req.session.usuario.usuario;
  const r = await pool.request()
    .input('u', sql.VarChar, usuario)
    .query('SELECT id, protocolo, titulo, status FROM chamados_tickets WHERE solicitante = @u AND status != \'Fechado\'');
  res.json(r.recordset);
});

2. Criar função em portalApi.js

async function listarMeusChamados(login) {
  const { data } = await api.get(`/api/chamados/meus?_whatsapp_login=${login}`);
  return Array.isArray(data) ? data : [];
}
module.exports = { ..., listarMeusChamados };

3. Adicionar comando no messageHandler.js

if (texto === 'chamados' || texto === '8') {
  const lista = await listarMeusChamados(sessao.login);
  const msg = lista.length
    ? lista.map(c => `#${c.protocolo} — ${c.titulo} (${c.status})`).join('\n')
    : 'Nenhum chamado aberto.';
  return await enviarMensagem(jid, `📋 *Chamados:*\n\n${msg}`);
}
